# Каталог расширений для Lampa

**Lampa** — приложение для просмотра (в т. ч. на ТВ и приставках). Расширения в неё ставятся **плагинами**: это обычные `.js`-файлы по ссылке.

**Этот репозиторий** — как раз один такой **каталог**: файл [`extensions.json`](extensions.json) — это список расширений (название, автор, описание, URL скрипта), который открывает **встроенный магазин** в Lampa. Скрипт [`store.js`](store.js) добавляет в настройки пункт этого магазина.

Часть плагинов лежит **здесь** и раздаётся с GitHub Pages; остальные записи в JSON просто указывают на чужие URL.

**Ссылки после публикации на Pages:**

- Каталог (то, что читает магазин): `https://lampaplugins.github.io/store/extensions.json`
- Подключить магазин в Lampa одной строкой: `https://lampaplugins.github.io/store/store.js`

---

### За три шага для пользователя

1. В Lampa добавьте плагином ссылку на **store.js** (URL из блока со ссылками выше).
2. В настройках откройте появившийся пункт магазина.
3. Выберите нужное расширение из списка.

В каталоге есть и **чужие** плагины (другие сайты и авторы) — ставьте только то, чему доверяете.

### Главные файлы репозитория

| Файл | Роль |
|------|------|
| [`extensions.json`](extensions.json) | Каталог: что показывает магазин в Lampa |
| [`store.js`](store.js) | Плагин входа в настройки → открыть этот каталог |

Локальные плагины из корня доступны как  
`https://lampaplugins.github.io/store/имя-файла.js`

### Контрибьюторам: новый плагин

1. Файл `.js` — в корень **или** внешний `link` только в JSON.
2. Добавьте блок в нужную категорию [`extensions.json`](extensions.json) (скопируйте соседний по формату).
3. Обновите список в `<details>` выше одной строкой.
4. Проверка: `python3 -m json.tool extensions.json > /dev/null`

Пример блока:

```json
{
  "name": "Название",
  "author": "@вы",
  "link": "https://…/plugin.js",
  "descr": "Что делает пользователю.",
  "version": "1.0.0"
}
```

Стиль кода — как у соседних файлов; после смены каталога иногда нужен повторный заход в магазин.
